Class: XMLHttpRequest

XMLHttpRequest()

This class implements HTTP requests. HTTP responses from these requests may be text or JSON converted to a Mildew Object as there is currently no Document or XML class. Requests may be synchronous or asynchronous.

All event handlers in this class except onreadystatechange receive an Object containing the following fields: "lengthComputable", a boolean indicating whether the total size of the download could be calculated before completion, "loaded", the number of bytes downloaded so far, and "total", the total size of the HTTP response body if known, otherwise 0. The "this" keyword may be used to refer to the XMLHttpRequest instance inside its event handler callbacks.

onloadend

Sets the event handler for the "loadend" event. This event is called when the HTTP request is either aborted, terminated due to error, or completed successfully.

onloadstart

onprogress

Gets the callback for the "progress" event or null if it has not been set.

onprogress

Sets the callback for the "progress" event. The progress event is called each time progress is made toward completion of the download.

onreadystatechange

Gets the callback for the "readystatechange" event or null if it has not been set.

onreadystatechange

Sets the callback for the "readystatechange" event. This event callback is called with no parameters but the "this" object is the XMLHttpRequest object associated with it. The readyState field can be checked for the following values: 1 indicates that the request has been opened with a call to open. 2 indicates that the headers have been received. 3 indicates that response body is loading. 4 indicates that the response is complete and the status field can be checked for HTTP status codes such as 200 for success or 404 for "not found."

ontimeout

Gets the callback for the "timeout" event if set.

ontimeout

Sets the callback for the "timeout" event. This event only occurs when the timeout property is set.

readyState

Returns the ready state, a number from 0-4. 0 Indicates that the request has not been sent. 1 indicates that the request has been opened with the open method. 2 indicates that the headers have all been received. 3 indicates that the response body is loading. 4 indicates that the request is finished.

response

This returns the Mildew value form of the response, or undefined if the readyState is not yet 4. So far only application/json responses are converted into a Mildew Object, otherwise the return value is the same as responseText.

responseText

Attempts to convert the bytes received as the response body to a UTF-8 string and return it. This may throw an error if the response is binary data that does not form a UTF-8 string. If the ready state is not yet 4, the return value is an empty String.

status

Once the status code is received from the server, this property is available. See https://developer.mozilla.org/en-US/docs/Web/HTTP/Status for a list of possible values

timeout

Gets the previously set timeout value in milliseconds.

timeout

Sets the timeout for all HTTP operations in milliseconds. This must be called after open otherwise an error will be thrown.

Overrides how the response property will be converted. For example, if an "application/json" response body was sent but this method is called with a "text/plain" parameter, the response property will be a String rather than Object. This method must be called before send to have any effect.
